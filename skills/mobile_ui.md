# 移动端开发规则

## 设计目标

这是一个工程巡检类 App，不是社交或消费类 App。
UI 应优先考虑：
- 快速录入
- 清晰结构
- 低学习成本
- 弱网/无网可用
- 单手操作友好

## 规则

1. 页面结构简单清晰
2. 单个检查项详情页底部固定保存按钮
3. 必填项明确标识
4. 重要状态用明显视觉层级区分
5. 减少复杂动画
6. 列表页优先展示完成度和异常数
7. 离线状态明显提示
8. 操作按钮文案直接，不要花哨
9. 优先适配 Android 使用习惯
10. 拍照、录音、定位入口始终清晰可见

## 检查项详情页必须包含

- 检查项名称
- 检查状态
- 是否存在问题
- 问题类型
- 严重程度
- 检查情况记录
- 建议措施
- 照片入口
- 语音入口
- 定位入口
- 附件入口
- 保存按钮

## 表单规则

1. issue_flag = false 时，隐藏问题类型和严重程度
2. 模板项若 supports_photo=false，则不显示拍照入口
3. 模板项若 supports_attachment=false，则不显示附件入口
4. 表单组件要可复用
