# 水库现场安全检查 App MVP 需求文档

## 1. 产品定位

这是一个基于规范附录A的水库现场安全检查与取证 App。

它不是普通拍照工具，而是一个“规范检查表驱动”的移动检查系统：

- 规范表是主线
- 检查项是基本单元
- 照片、语音、定位、附件是证据
- 最终自动形成检查表、问题清单、照片附表

## 2. 业务背景

现有现场检查工作中，技术人员通常需要：

1. 到现场检查
2. 拍摄大量照片
3. 回办公室后回忆每张照片对应部位
4. 整理检查情况
5. 再写入巡查表或报告

主要问题：
- 照片与部位容易混淆
- 整理耗时长
- 记录不规范
- 现场信息容易遗忘
- 成果输出重复劳动多

## 3. 产品目标

MVP 目标：

1. 将检查工作按 A.1～A.8 章节结构化
2. 支持每个检查项快速录入
3. 支持照片、语音、定位等证据挂接
4. 支持离线使用
5. 支持导出规范化成果

## 4. 用户角色

### 4.1 现场检查人员
- 创建检查任务
- 进入章节
- 填写检查项
- 拍照、录音、定位
- 提交记录

### 4.2 项目负责人/内业人员
- 复核记录
- 查看异常项
- 导出表格和附表
- 用导出成果继续编写正式报告

## 5. 规范章节范围

### A.1 现场安全检查基本情况
- 水库名称及基本情况描述
- 枢纽工程主要建筑物
- 水库防洪保护对象
- 检查时间
- 天气
- 检查时库水位
- 检查时库容
- 检查人员
- 现场检查发现的主要问题描述

### A.2 挡水建筑物现场检查情况——土石坝
按坝顶、坝体、坝基、坝肩、下游地面、近坝库岸等组织

### A.3 挡水建筑物现场检查情况——混凝土坝与浆砌石坝
按坝顶、坝体、坝基、坝肩、下游地面、近坝库岸等组织

### A.4 泄水建筑物现场检查情况——溢洪道

### A.5 泄水建筑物现场检查情况——溢（泄）洪隧洞

### A.6 输（引）水建筑物现场检查情况

### A.7 管理设施现场检查情况

### A.8 水库上下游现场检查情况

## 6. MVP 范围

### 必做
- 登录
- 创建检查任务
- 按模板树展示章节和检查项
- 填写检查状态、问题、记录、建议
- 绑定照片、语音、定位、附件
- 本地离线保存
- 异常项汇总
- 导出检查表、问题清单、照片附表

### 暂不做
- 图像识别裂缝
- 多组织商业化权限
- 高级 GIS
- 实时协同
- 问题闭环整改系统
- 完整报告全文自动生成

## 7. 页面结构

1. 登录页
2. 任务列表页
3. 任务详情页
4. 章节列表页
5. 检查项列表页
6. 检查项详情页
7. 异常项汇总页
8. 导出页

## 8. 检查项统一结果字段

每个检查项统一具备：

- check_status
- issue_flag
- issue_type
- severity_level
- check_record
- suggestion
- photo_count
- audio_count
- video_count
- gps_lat
- gps_lng
- location_desc
- attachment_count

## 9. 录入原则

- 优先快速录入
- 优先选择项 + 语音
- 必填项尽量少
- 异常项必须补全描述和建议
- 支持事后编辑

## 10. 成果输出

### 10.1 规范检查表
按 A.1～A.8 导出

### 10.2 问题清单
提取 issue_flag = true 的记录

### 10.3 照片附表
按章节和检查项归类展示证据

## 11. MVP 验收标准

1. 可创建任务
2. 可按章节和检查项检查
3. 可记录文本、照片、语音、定位
4. 可离线保存并同步
5. 可导出规范成果
6. 可明显减少人工整理时间
